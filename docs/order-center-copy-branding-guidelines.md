# 주문센터 용어/브랜딩/카피 기준 (v1)

- 문서 버전: `v1 (초안)`
- 작성일: `2026-02-23`
- 목적: 주문센터 전체 페이지의 용어/브랜딩/카피 기준을 먼저 고정하여, 이후 핫픽스/시스템 기능 추가/전페이지 통일성 작업의 기준선으로 사용한다.

## 1. 기본 원칙

## 1.1 서비스 성격 표현 원칙

주문센터는 사용자가 사양/구성을 입력하고, 운영팀 확인 후 결제 링크 또는 상담 연결이 진행되는 구조다.

따라서 용어는 아래처럼 분리한다.

1. 사용자 행동(메인 흐름): `주문`
2. 금액/계산 결과: `견적`
3. 최종 제출 액션: `주문 요청 전송`
4. 운영 처리 결과 안내: `접수`, `확인 후 안내`

## 1.2 문구 톤 원칙

1. 짧고 명확하게 쓴다.
2. 같은 의미는 같은 표현을 쓴다.
3. 버튼 라벨은 동사형으로 통일한다.
4. 페이지 제목과 H1은 같은 의미/표현으로 맞춘다.

## 2. 용어 체계 (확정안)

## 2.1 핵심 용어 정의

1. `주문`: 사용자가 사양을 선택/구성하고 요청을 제출하는 전체 행위
2. `견적`: 계산된 금액/상담안내 여부가 포함된 가격 정보
3. `주문 요청`: 최종 제출 행위 (메일 전송 포함)
4. `상담 안내`: 자동 가격 계산 대신 상담이 필요한 상태
5. `구성`: 시스템 페이지에서 모듈 조합/레이아웃을 의미하는 용어

## 2.2 금지/지양 표현

1. 같은 위치에서 `주문`과 `견적`을 혼용하는 표현
2. 의미가 모호한 표현 (`전송`, `확인`만 단독 사용)
3. 붙여쓰기 불일치 (`합판담기`, `도어담기`) 형태

## 3. 페이지별 제목/H1 기준

원칙:

1. 브라우저 `<title>`과 H1(`.title-main`)을 동일하게 맞춘다.
2. 형식은 `OOO 주문하기`로 통일한다.

### 3.1 페이지 기준 문구

1. `index`
   - `<title>`: `가구랑 주문 센터`
   - `H1`: `가구랑 주문 센터`
2. `board`
   - `<title>`: `합판 주문하기`
   - `H1`: `합판 주문하기`
3. `door`
   - `<title>`: `도어 주문하기`
   - `H1`: `도어 주문하기`
4. `top`
   - `<title>`: `상판 주문하기` (현재 `상판 견적 계산기`에서 변경)
   - `H1`: `상판 주문하기`
5. `system`
   - `<title>`: `시스템 수납장 주문하기`
   - `H1`: `시스템 수납장 주문하기`

## 4. 공통 버튼 라벨 기준

## 4.1 단계/네비게이션 버튼

1. 이전: `이전`
2. 다음: `다음`
3. 랜딩 복귀: `이전` (현행 유지 가능) 또는 `주문센터로`
   - 1차는 현행 유지, 2차에서 선택 가능
4. 초기화/완료 후 복귀: `처음으로 돌아가기`

## 4.2 최종 제출 버튼

기준 라벨:

1. `주문 요청 전송`

이유:

1. 단순 `주문 전송`보다 실제 운영 프로세스(확인 후 결제/상담)를 더 정확히 전달함

## 4.3 액션 카드(담기) 버튼

원칙:

1. 공백 포함 `{대상} 담기`
2. 시스템 페이지처럼 “제품 단품”이 아닌 경우는 `구성 담기` 사용

표준 라벨:

1. 합판 페이지: `합판 담기`
2. 도어 페이지: `도어 담기`
3. 상판 페이지: `상판 담기`
4. 시스템 페이지: `구성 담기`

## 5. 요약 패널/완료 문구 기준

## 5.1 우측 요약 패널 제목

현행:

- `주문하실 견적서를 확인해주세요.`

표준안:

- `주문 예정 항목을 확인해주세요.`

이유:

1. `주문`과 `견적서` 혼용을 줄임
2. 요약 패널의 실제 역할(항목 확인)에 더 정확함

## 5.2 완료 메시지

완료 타이틀 표준안:

- `주문 요청이 접수되었습니다.`

완료 본문 표준안:

- `내용 확인 후 기재된 연락처로 결제 링크 또는 상담 주문 연결을 도와드리겠습니다.`

## 5.3 빈 상태/에러 메시지 톤

원칙:

1. 무엇이 필요한지 바로 말한다.
2. 가능하면 행동 유도 문구를 포함한다.

예시:

1. `담긴 항목이 없습니다. 주문을 담아주세요.`
2. `주문 전송 중 오류가 발생했습니다. 다시 시도해주세요.`

## 6. 시스템 페이지 신규 기능 라벨 기준 (확정안)

## 6.1 타입/섹션 기능

1. 섹션 그룹 제목: `레이아웃 타입 및 섹션 설정`
2. 타입 라벨: `레이아웃 타입`
3. 타입 선택지:
   - `ㅣ자`
   - `ㄱ자`
   - `ㄷ자`
   - `ㅁ자`
4. 섹션 입력 라벨:
   - `섹션1 길이 (mm)`
   - `섹션2 길이 (mm)` ...
5. 상태 라벨:
   - `정상 견적 가능`
   - `상담 안내 대상`
   - `구성 불가`

## 6.2 상담 안내 기준 관련 문구

1. `가장 낮은 높이가 2700mm 이상이면 상담 안내로 처리됩니다.`
2. `섹션 길이가 8000mm 이상이면 상담 안내로 처리됩니다.`

주의:

1. 위 문구는 입력 차단이 아니라 `가격 상담 전환`을 명시해야 한다.

## 6.3 모듈 모달 정면 미리보기

1. 섹션 제목: `정면 미리보기`
2. 보조 문구(선택): `선반 갯수와 구성품이 실시간으로 반영됩니다.`
3. 요약 라벨 예시:
   - `선반 3개`
   - `옷봉 1개`
   - `구성품 없음`

## 7. 브랜딩 표현 기준 (1차/2차 분리)

## 7.1 1차 배포 기준 (최소 통일)

1. 페이지 제목/H1/버튼/완료문구 통일
2. 오타/띄어쓰기/문장 톤 통일
3. 시스템 신규 기능 라벨을 본 문서 기준으로 적용

## 7.2 2차 배포 기준 (고도화)

1. 랜딩 페이지 브랜드 존재감 강화(로고/메타 정보)
2. 섹션/배지/요약 패널의 카피 위계 정리
3. 영문 서브타이틀 표현 규칙 정리 (`Order`, `Order Center` 톤 통일)

## 8. 빠른 핫픽스 대상 (3단계 선반영용)

1. `상판 견적 계산기` -> `상판 주문하기` (`top.html` title)
2. `합판담기` -> `합판 담기` (`board.html`)
3. `도어담기` -> `도어 담기` (`door.html`)
4. `주문 전송` 버튼 -> `주문 요청 전송` (전 페이지)
5. 요약 패널 제목 문구 통일 (전 페이지)
6. 도어 페이지 문구 오타 수정 (`도어이나`, 띄어쓰기)

## 9. 적용 우선순위

1. 시스템 페이지 신규 기능 관련 라벨 (즉시)
2. 전페이지 버튼/제목/완료문구 통일 (핫픽스)
3. 랜딩/브랜딩 고도화 (2차)
