# 주문센터 개선 범위 및 배포 계획 (1차/2차)

- 문서 버전: `v0.1 (초안)`
- 작성일: `2026-02-23`
- 목적: 시스템 페이지 신규 기능과 전체 개선(브랜딩/모바일/접근성/통일성)의 범위를 먼저 확정하고, 배포 단위를 `1차/2차`로 분리한다.

## 1. 프로젝트 목표

이번 작업의 목표는 아래 2가지를 동시에 달성하는 것이다.

1. 시스템 페이지의 실무 사용성을 높이는 기능 추가
2. 주문센터 전체 페이지의 UI/UX 완성도와 일관성 개선

## 2. 범위 확정 (전체)

## 2.1 포함 범위 (In Scope)

### A. 시스템 페이지 신규 기능 (핵심)

1. 타입 선택 + 섹션별 사이즈 입력 + 가용범위 제약 기능
2. 각 모듈 옵션 모달 내 정면 미리보기(선반갯수, 구성품) 기능

### B. 시스템 페이지 품질 개선 (신규 기능 연계)

1. 타입/섹션 사양의 메인 미리보기 반영
2. 타입/섹션 제약의 모듈 추가/수정 제약 연동
3. 타입/섹션 사양의 견적 상세/주문완료/메일 반영
4. 시스템 페이지 신규 기능 관련 카피/라벨 정리

### C. 주문센터 전체 개선 범위 (공통)

1. 브랜딩 통일성 개선
2. 모바일 레이아웃/입력 UX 개선
3. 접근성 개선(모달/키보드/포커스/모션)
4. 페이지 간 용어/버튼/요약 패널 통일성 개선

## 2.2 제외 범위 (Out of Scope, 이번 배포 기준)

1. 완전 자유형 도면 편집기(드래그 기반 레이아웃 에디터)
2. 3D 미리보기
3. 서버/백엔드 주문 시스템 연동 변경
4. 결제 시스템 구조 변경
5. 디자인 시스템 전면 재구축

## 3. 시스템 신규 기능 범위 확정 (반영된 결정사항)

## 3.1 타입 범위 (확정)

- 사용 타입:
  - `ㅣ자`
  - `ㄱ자`
  - `역ㄱ자`
  - `ㄷ자`
  - `ㅁ자`
- 제외 타입:
  - `ㅣㅣ자` (사용하지 않음, 제거)
- 내부 키 제거:
  - `i_double` (사용하지 않음, 제거)

권장 내부 키(현재 기준):

1. `i_single` (`ㅣ자`)
2. `l_shape` (`ㄱ자`)
3. `rl_shape` (`역ㄱ자`)
4. `u_shape` (`ㄷ자`)
5. `box_shape` (`ㅁ자`)

## 3.2 섹션 길이 입력 정책 (확정)

1. 섹션 길이 최소값: `460mm`
2. 섹션 길이 최대값: `없음` (정책상 상한 없음)
3. 상담안내 전환 기준:
   - 한 섹션 길이가 `8000mm 이상`이면 가격을 `상담안내`로 처리

## 3.3 높이 입력 정책 (확정)

1. 가장 낮은 높이가 `2700mm 이상`이면 가격을 `상담안내`로 처리

정책 의미:

1. 위 조건은 `invalid(구성불가)`가 아니라 `consult(상담안내)` 기준으로 사용한다.
2. 입력 자체는 허용하되, 견적/요약/전송 단계에서 상담안내 가격으로 처리한다.

## 3.4 체크포인트 반영 기준 (현재 확정분)

필수 체크포인트:

1. `미리보기에 보여야 함`

적용 방식:

1. 타입/섹션 사양은 메인 탑뷰 미리보기에 반드시 반영
2. 모듈 옵션 정보(선반갯수/구성품)는 각 모듈 모달 정면 미리보기에 반드시 반영

## 4. 전체 개선 범위 세부 정의 (브랜딩/모바일/접근성/통일성)

## 4.1 브랜딩 개선 범위

1. 용어 체계 통일 (`주문`, `견적`, `전송`)
2. 페이지 타이틀/H1/버튼 라벨 톤 통일
3. 완료 메시지/안내 메시지 톤 정리
4. 필요 시 랜딩 페이지 브랜드 존재감 강화(로고/메타 정보)

## 4.2 모바일 개선 범위

1. 폼 레이아웃(라벨/입력 필드) 모바일 최적화
2. 하단 고정 요약 패널(`estimate-panel`) 동작 개선
3. 모달 크기/레이아웃 모바일 최적화
4. 긴 콘텐츠(개인정보/모달 목록) 스크롤 인지성 개선

## 4.3 접근성 개선 범위

1. 모달 ARIA 속성(`role="dialog"`, `aria-modal`, `aria-labelledby`)
2. 키보드 조작 (`Esc` 닫기, 포커스 복귀, 포커스 흐름)
3. 포커스 스타일 개선
4. 모션 감쇠(`prefers-reduced-motion`) 대응

## 4.4 통일성 개선 범위

1. 버튼 라벨 표기 규칙 통일 (`합판 담기`, `도어 담기`, `상판 담기`, `견적 담기`)
2. 요약 패널 정보 정책 통일(총액만/자재비/배송비 표시 기준)
3. 페이지 간 모달 구조/푸터 버튼 우선순위 통일
4. 에러/안내 메시지 위치 및 톤 통일

## 5. 배포 단위 확정 (1차 / 2차)

## 5.1 1차 배포 범위 (핵심 기능 + 사용성 기반)

목표:

1. 시스템 신규 기능을 실사용 가능한 수준으로 제공
2. 신규 기능 구현에 필요한 공통 기반을 먼저 안정화

포함 범위:

1. 시스템 기능 A: 타입 선택 + 섹션 입력 + 가용범위 제약
2. 시스템 기능 B: 모듈 옵션 모달 정면 미리보기
3. 시스템 페이지 견적/주문완료/메일 반영
4. 시스템 페이지 관련 카피/라벨 정리
5. 시스템 기능에 직접 영향 있는 모바일/모달 최소 개선
6. 전페이지 빠른 핫픽스(오타/카피 오류/CSS 오타 등)

포함하는 공통 개선 (최소 필수):

1. 모달 접근성 최소셋(ARIA + Esc 닫기 + 포커스 복귀)
2. 시스템 페이지 중심 모바일 폼/모달 레이아웃 개선

제외(2차로 이관):

1. 전페이지 접근성 고도화 전부
2. 브랜딩 정교화(로고 활용/랜딩 강화 등)
3. `board.js`/`door.js` 공통화 리팩토링

## 5.2 2차 배포 범위 (전페이지 통일성 + 품질 고도화)

목표:

1. 시스템 기능 추가 이후 전체 주문센터 품질을 균형 있게 끌어올림
2. 유지보수성 개선

포함 범위:

1. 브랜딩/카피 전페이지 통일 정리
2. 모바일 UX 전페이지 보강
3. 접근성 개선 전페이지 확대 적용
4. 요약 패널 정보 정책 통일
5. 공통 모달/완료 UI/주소검색/개인정보 처리방침 구조 정리
6. `board.js`/`door.js` 중복 로직 공통화
7. 통합 QA/회귀 테스트 강화

## 6. 권장 작업 순서 (이번 결정 기준)

1. `범위/배포 기준 확정` (현재 문서)
2. `S-00 문서 업데이트` (타입 제거, 상담안내 기준 반영)
3. `시스템 기능 A/B 구현` (시스템 페이지 우선)
4. `시스템 페이지 QA`
5. `1차 배포`
6. `전페이지 공통 개선/통일성 작업`
7. `2차 배포`

## 7. 1차/2차 릴리즈 게이트 (간단 기준)

## 7.1 1차 배포 게이트

1. 시스템 페이지에서 타입/섹션 입력이 동작한다.
2. 타입/섹션 정보가 메인 미리보기에 반영된다. (체크포인트 충족)
3. 모듈 모달 정면 미리보기가 동작한다.
4. 견적/주문완료/메일에 신규 사양 정보가 반영된다.
5. 기존 시스템 페이지 핵심 플로우 회귀 없음

## 7.2 2차 배포 게이트

1. 전페이지 카피/용어 규칙 적용 완료
2. 모바일 레이아웃 주요 페이지 검수 완료
3. 모달 접근성/키보드 흐름 검수 완료
4. 중복 리팩토링 후 회귀 테스트 완료

## 8. 현재 오픈 이슈 (다음 단계에서 처리)

1. 시스템 기능 A의 UI 세부 구조(타입 선택 UI 형태, 섹션 입력 레이아웃) 확정
2. `S-01` 구현 범위 고정 (상태 모델 + shape 키 정리 중심)
