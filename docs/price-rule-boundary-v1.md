# Price Rule Boundary v1

## 1. 목적
- 금액 계산, 상담안내 전환, 검증 시점을 문서로 고정해 페이지별 규칙 불일치를 방지한다.

## 2. 고정 원칙
1. 검증 시점은 `담기 단계`로 고정한다.
2. 가공 토글 OFF 시 가공 선택/세부 입력은 즉시 초기화한다.
3. 상담품목은 UI에서 `상담 안내`로 표기한다.
4. 상담품목의 payload 가격 필드는 `null`, `displayPriceLabel: "상담안내"`로 표현한다.

## 3. 계산 경계
1. `board`, `door`는 배송비를 포함한 `grandTotal`을 계산한다.
2. `top`은 현재 배송비를 `0`으로 처리한다.
3. 상담품목이 포함된 주문은 합계에 `상담 필요 품목 미포함` 라벨을 함께 표기한다.

## 4. 옵션/가공 ID 정책
1. 전역 유니크 접두 규칙을 사용한다.
2. 예시
- board 옵션: `board_*`
- door 옵션: `door_*`
- top 옵션: `top_*`
- 공통 가공: `proc_*`

## 5. 운영 체크리스트
1. 신규 옵션/가공 추가 시 접두 규칙을 지켰는지 확인
2. 담기 버튼에서 검증 차단이 동작하는지 확인
3. 상담품목 payload가 `null + 상담안내` 규칙을 지키는지 확인
