# 시스템 미리보기 모듈 사이 삽입 1차 회귀 체크리스트

## 1. 범위

- 대상 화면: `system.html`
- 대상 기능: `일반 모듈 - 일반 모듈 사이 삽입(+)`
- 포함 플로우:
  - 끝점 추가
  - 모듈 사이 삽입
  - 코너 삭제
  - Undo/Redo

## 2. 사전 조건

- 시스템 페이지 진입 후 기본 옵션(선반/포스트바 컬러, 높이) 선택 완료
- 미리보기가 정상 렌더되는 상태
- 브라우저 캐시 비움 또는 강력 새로고침 후 확인

## 3. 핵심 회귀 시나리오

### 시나리오 A: 직선 구성에서 사이 삽입

1. 일반 모듈 2개를 연속으로 추가한다.
2. 두 모듈 사이 공유 기둥 위치에 `+` 버튼이 노출되는지 확인한다.
3. 해당 `+`를 클릭한다.
4. 일반 모듈이 정확히 중간에 1개 추가되는지 확인한다.
5. 삽입 직후 새 모듈 옵션 모달이 열리는지 확인한다.

기대 결과:

- 중간 삽입 후 모듈 순서가 `기존-신규-기존`으로 유지된다.
- 기존 뒤 모듈이 튀지 않고 기존 진행 방향을 유지한다.

### 시나리오 B: ㄱ자/ㄷ자에서 사이 삽입 + 코너 삭제

1. ㄱ자 또는 ㄷ자로 일반/코너 혼합 구성을 만든다.
2. 일반-일반으로 직접 연결된 구간에서만 `사이 삽입 +`가 노출되는지 확인한다.
3. 코너 인접/포함 구간에서는 삽입 버튼이 노출되지 않는지 확인한다.
4. 일반-일반 구간에서 사이 삽입을 1회 수행한다.
5. 이후 코너를 삭제한다.

기대 결과:

- 삽입된 일반 모듈이 있어도 코너 삭제 후 일반 모듈 위치가 비정상 이동하지 않는다.

### 시나리오 C: Undo/Redo

1. 일반 모듈 2개 구성에서 사이 삽입 1회를 수행한다.
2. `Undo` 1회: 삽입 이전 상태로 정확히 복귀하는지 확인한다.
3. `Redo` 1회: 삽입 상태로 정확히 재적용되는지 확인한다.
4. 다시 코너 추가/삭제를 섞어 `Undo/Redo`를 반복한다.

기대 결과:

- 각 단계에서 모듈 수/순서/앵커 위치가 일관되게 복원된다.
- 예상치 못한 모듈 점프나 중복/유실이 발생하지 않는다.

## 4. 비노출/가드 확인

1. 모듈이 0개 또는 1개일 때 사이 삽입 버튼이 노출되지 않아야 한다.
2. 한 부모 endpoint에 자식이 2개 이상 연결된 분기점에서는 사이 삽입 버튼이 노출되지 않아야 한다.
3. 코너가 포함된 연결점에서는 사이 삽입 버튼이 노출되지 않아야 한다.
